Phần I, Phân biệt git vs githup

    Git là gì ?

  • Git còn được gọi là Distributed Version Control System (DVCS) hay VCS, là một hệ thống quản lý phiên bản phân tán ra đời vào năm 2005
  • Git là một hệ thống quản lý phiên bản mã nguồn. Nó cho phép lưu trữ, theo dõi và quản lý lịch sử thay đổi của mã nguồn.
  • Git là công cụ phổ biến được cài đặt trên máy tính cá nhân của người dùng để làm việc với các repository (kho chứa mã nguồn).

    • Githup là gì

  • GitHub là một dịch vụ lưu trữ mã nguồn (source code) trực tuyến dựa trên nền tảng Git. Nó cung cấp một nền tảng cho các nhà phát triển phát triển, quản lý và theo dõi mã nguồn của dự án phần mềm. Dịch vụ này cho phép các nhóm phát triển làm việc cùng nhau trên các dự án phần mềm từ xa và theo dõi lịch sử của mã nguồn.

    • Lệnh Git cở bản

  • git clone ; là một lệnh trong hệ thống Git cho phép các lập trình viên sao chép một kho chứa từ xa (Remote Repo) về máy tính cá nhân (Local)
  • git status ; Kiểm tra trạng thái của file đã/có thay đổi trong thư mục.
  • git commit -m ""; comment lại nội dung hoặc định nghĩa rõ nội dung của file đó
  • git add . ; thêm tất cả file trong thư mục dấu . nằm sau lệnh git add nghĩa là lấy tất cả các file có trong thu mục
  • git push ; git push được sử dụng để đẩy (push) các commit và thay đổi từ máy tính của bạn lên một remote repository

Phần II, Vai trò Html, CSS, JavaScript ?

HTML – Xây dựng cấu trúc và định dạng các siêu văn bản.

CSS – Định dạng các siêu văn bản dạng thô tạo ra từ HTML thành một bố cục website, có màu sắc, ảnh nền,….

Javascript – Tạo ra các sự kiện tương tác với hành động của người dùng (ví dụ như là chat, update nội dung, hiệu ứng slide).

Phần III, JavaScript là gì?

JavaScript ? Javascript là một ngôn ngữ lập trình kịch bản dựa vào đối tượng phát triển có sẵn hoặc tự định nghĩa. Javascript được sử dụng rộng rãi trong các ứng dụng Website.

    Cách khai báo hàm trong JS

  • Đầu tiên ta có thể khai báo 1 hàm vô cùng đơn giản với từ khóa function, theo sau nó là tên của hàm đó cùng với 1 hoặc nhiều tham số (parameters) được tập hợp lại trong dấu ngoặc tròn (), cuối cùng là phần thân hàm sẽ được nhận biết thông qua cặp ngoặc nhọn {}.
    function name_function(tham số) {content_code}
    VD. function example() {console.log("Đây là ví dụ");} _or_ VD. function isEven(num) {return num % 2 === 0;}

    • Gán function vào một biến : Chúng ta khai báo một biến, gán giá trị của biến này là một function

  • const foo = function () {console.log(‘vui lap trinh’)} lưu ý là không dùng function trước khi khai báo

    • Hàm mũi tên (Arrow Function)

  • Khai báo Arrow Function bằng dấu => mũi tên ... VD. let sum = (a, b) => a + b;

    • Data types

  • Trong JS có 6 kiểu dữ liệu cơ bản : Number, Boolean, String, Null, Undefined, Object
  • Đọc tại webcoban.vn hoặc viblo.asia để hiểu kỹ hơn

Hạn chế của khi sử dụng JavaScript? không dõ ràng , không chặt chẽ trong chương trình . VD. nếu trương trình của bạn báo lỗi và khi bạn debug thì với kiểu dữ liệu mập mờ của JS thì toang nha !

Phần III, Typescript là gi?

    Do sự hạn chế của javascript nên đã có sự xuất hiện của ngôn ngữ Typescript để bổ sung lại các thiếu sót đó . Vậy Typescript là gì ?

    Typescript ?

  • TypeScript là một ngôn ngữ lập trình được phát triển và duy trì bởi Microsoft , là một ngôn ngữ mã nguồn mở của JavaScript .
  • TypeScript được xem là một phiên bản nâng cao hơn của JavaScript vì nó được thiết kế thêm nhiều chức năng tiện lợi hơn, cải tiến hơn từ những điểm yếu của JavaScript như các lớp hướng đối tượng và Static Structural typing, bên cạnh đó TypeScript còn có thể hoạt động rộng rãi cho các ứng dụng của ngôn ngữ Angular2 và Nodejs.
  • Typescript Không phải là một ngôn ngữ mới
  • Typescript nó chỉ là một phiên bản mở rộng của JavaScript
  • TypeScript sinh ra chỉ để cho các lập trình viên chứ không cho trình duyệt

    • Kiểu dữ liệu trong typescript

  • Primitive (kiểu nguyên thủy) : Number, String, Boolean, void, Undefined, Null ...
  • Vậy làm sao để chạy được TypeScript chạy được trên trình duyệt ?
  • Để TypeScript(TS) chạy được trên trình duyệt thì nó phải chuyển đổi thành mã JS và JS thì sẽ chạy được trên trình duyệt

    • Vậy làm sao chuyển đổi từ (TS) thành (JS)

  • Giả sử ta có file là index.js ta run nó trên trình duyệt sau đó từ trình duyệt ta chon inspect để open cửa sổ devtool nên rồi tìm và chọn console và chạy code trên đó nếu làm vậy sẽ rất mất công và thời gian
  • Thay vì làm như trên ta tạo một sever (Node.js) rồi chạy file index.js trên nó sever đó

Phần IV, Chạy typescript = terminal trên Visua Studio Code

    Các bước và lệnh để chạy Typescript

  • B1 : click chuột trái vào thư mục Demo 1 chọn copy path(lấy đường dẫn của thư mục đó)
  • B2 : open terminal chọn New terminal nó sẽ hiên thị như sau "PS D:\Railway_82_Repository>"
  • B3 : dùng lệnh cd + 'đường dẫn của thu mục đó'
  • B4 : sau B3 ta có cú pháp sau : PS D:\Railway_82_Repository> cd 'D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1'

      Trong đó :

    • PS D:\Railway_82_Repository> (terminal)
    • cd (chuyển thư mục)
    • ký tự '' ()
    • D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1 (là đường dẫn của thu mục)
    • Waring : nếu viết như này cd D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1 mà không đặt trong dấu ngoặc '' thì sẽ báo lỗi vì nó không hiểu đây là gì vì có các dấu cách
  • B5 : khi hiển thị như này PS D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1> là ok
  • B6 : gõ lệnh ls để hiển thị danh sách các file có trong thư mục ... VD. PS D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1> ls
  • B7 : sau B6 ta có lệnh tsc + .\Demo1.ts (viết theo kiểu shorthand .\ + tag)...
    VD. PS D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1> tsc .\Demo1.ts
  • B8 : Nếu run ở B7 mà nó bị lỗi có nghĩa là do biến ở môi trường chưa đúng .
  • B9 : Sửa lỗi B7 như sau ta copy lện này Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy Unrestricted vào và run tiếp
  • B10 : Dùng lệnh node .\Demo1.js để in ra kết quả ... VD. PS D:\Railway_82_Repository\1. Typescript + JS Extra\Demo 1> node .\Demo1.js để hiển thị Đoạn mã để in ra màn hình trong JS console.log("fuck me"); result fuck me (^_^)

    Tại Node.js chọn vào download rồi cài nodejs máy tính sau đó cứ next theo mặc định

  • Sau lệnh npm install -g typescript ta dùng lệnh tsc --version để kiểm tra xem cài đặt được chưa
